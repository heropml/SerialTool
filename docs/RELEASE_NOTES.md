# CommTool v1.1.6（自动应答·多步状态机）

串口 + 网络一体调试工具。本版给**自动应答**加上**多步状态机**：把多条规则串成按帧序列推进的握手 / 会话，并把状态推进、并发与边界处理一并做扎实。

## ✨ 新功能 / 改进

- **多步状态机** —— 弹窗新增「状态机」框（启用 / 初始状态 / 当前状态实时显示 / 重置）。每条规则可选两个字段：
  - **仅状态（when）** —— 只有当前状态等于它（可逗号分隔多个、状态名可含空格、留空 = 任意状态 / 通配）时该规则才命中；
  - **跳转（goto）** —— 该规则应答发出后，把当前状态切到它。

  用它实现「先收 A 才允许应答 B」「不同握手阶段回不同帧」等按序流程。每行 when/goto 两列仅在状态机启用时显示，不用时界面不变。
- **测试器同步** —— 离线规则测试器显示当前状态、命中后将跳转到的状态，以及「内容命中但状态不符 → 不会应答」的提示（预览 = 实发、不发送、不推进状态）。
- **稳健的状态推进** —— goto 仅在应答**真正发出**后才推进（延时未到 / 连接断 / TCP 无客户端 / 底层发送失败 / 故障注入丢包 / 坏 hex 都不推进），主机重试仍能命中当前状态。
- **并发序列化** —— 带 goto 的应答在整条（含多段 `|` + 段间延时）发完前，后续到达的帧进有界 FIFO（≤256），释放后按收帧顺序处理，杜绝随机延时下多任务乱序推进状态。
- 连接断开 / 重连 / 总开关切换 / 配置导入 / 手动重置会把状态机拉回初始；**编辑规则不会打断进行中的握手**。三语 i18n 同步；无新依赖；状态机默认关闭，关闭时行为与 v1.1.5 完全一致。
