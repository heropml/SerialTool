# CommTool v1.1.7（自动应答·Modbus RTU 从机）

串口 + 网络一体调试工具。本版给**自动应答**加上 **Modbus RTU 从机**模式：开启后整条引擎作为从机，从机地址匹配且 CRC 正确就按功能码从内置寄存器表自动组装标准响应回发，让你不接真实设备也能把上位机 / 主站调通。

## ✨ 新功能 / 改进

- **Modbus RTU 从机** —— 自动应答弹窗顶栏新增「Modbus 从机」按钮打开配置：**启用 + 从机地址 + 寄存器表**（按「空间 · 起始地址 · 值」配置初值，地址支持 `0x` / 十进制，线圈用 0/1）。开启后程序作为从机自动应答主机读写：
  - 读：`01` 线圈 / `02` 离散输入 / `03` 保持寄存器 / `04` 输入寄存器
  - 写：`05` 单线圈 / `06` 单寄存器 / `0F` 多线圈 / `10` 多寄存器（写功能码会改运行态寄存器，可被随后读回）
  - 非法功能码 / 越界地址 / 非法数据 自动回**异常响应**（`0x80|功能码` + 异常码）
- **稳健组帧** —— RTU 无帧头，按「功能码长度 + CRC 自洽」双重判定切帧、跨包缓冲；CRC 不符逐字节重同步，杜绝串口噪声 / 错位后永久失步。
- **精确回请求方** —— 作为 TCP 服务器带多客户端时，响应精确回发给请求的那个客户端（每客户端独立半包缓冲，断开自动清）。
- **故障注入仍生效** —— 全局丢包 / 错 CRC / 错长度照样作用于 Modbus 响应，可压测主站的重传与容错。
- 开启 Modbus 时规则 / 状态机 / 帧头组帧让位，弹窗自动收起这些区块、只留故障注入与醒目提示条，消除「看着启用却不响应」。
- 修复**暗色主题**下复选框看不出选中 / 未选中（加 `::indicator` 描边 / 主色填充样式）。三语 i18n 同步；新增 `tests/test_modbus_slave.py`（仓库首个自动化测试）；无新依赖；Modbus 默认关闭，关闭时行为与 v1.1.6 完全一致。
