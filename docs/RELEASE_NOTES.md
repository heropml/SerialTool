# CommTool v1.1.8（自动应答·位掩码 / 字段级匹配）

串口 + 网络一体调试工具。本版把**自动应答**的「匹配」做到**子字节粒度**：除了整字节 `??` 通配，现在还能按**半字节 / 位 / 字段**匹配，适配带标志位、子字段的协议帧。

## ✨ 新功能 / 改进

- **位掩码 / 字段级匹配** —— 「匹配」从「逐字节精确 / 整字节通配」升级为**逐字节 `(值, 掩码)` 比对**，命中需 `(收到字节 & 掩码) == (值 & 掩码)`：
  - **整字节**：精确 `AB`；通配 `??` / `XX`（不变）
  - **半字节通配**：`A?`（高 4 位 = A、低 4 位任意）、`?5`（低 4 位 = 5），`X` 同 `?`
  - **位级掩码**：`b:` + 8 位 `0/1/x`，`x` = 该位不关心，如 `b:1xxxxxx1` 只看最高位与最低位
  - 混写：`AA b:1001xxxx ?5` = 字节0 须 `AA`、字节1 高 4 位 `1001` 低 4 位任意、字节2 低 4 位 `5`
- **字段级** —— 「某偏移处的子字段 == X」用通配填充到该偏移 + 「前缀」模式表达（如第 3 字节 bit0 须为 1：匹配 `?? ?? ?? b:xxxxxxx1`、模式「前缀」）。
- **向后兼容** —— 空格仅作分隔，纯 HEX 跨空格拼接（`A B`=0xAB）；整字节精确 / 通配解析与改前**逐字节等价，旧规则零行为变化**。三种匹配模式（包含 / 相等 / 前缀）对掩码同样生效；位掩码仅 HEX 模式有意义。
- 实时匹配与**离线规则测试器**共用同一套解析，自动获得新能力；三语 i18n（占位符 + 帮助补「位 / 半字节掩码」一节）；新增 `tests/test_match_mask.py`（17 项，含向后兼容 / 坏格式 / 三模式）；无新依赖。
