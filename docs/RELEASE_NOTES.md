# CommTool v1.2.0（Modbus 主机轮询）

串口 + 网络一体调试工具。继自动应答的 Modbus RTU **从机**之后，本版补上 **主机（master / poll）**：把本工具当 Modbus 主机，按每行设定的周期轮询从机寄存器 / 线圈并实时显示结果——主机联调、查表、巡检从此不用再手敲请求帧。

## ✨ 新功能 / 改进

- **Modbus 主机轮询** —— 数据区标题栏新增「Modbus 主机」按钮，弹窗每行配置：
  - `名称 / 从机ID / 功能码 / 起始地址 / 数量(读) 或 写值(写) / 周期ms` → 实时 **值（十进制 + 十六进制）** + **状态**。
  - 读 **01** 线圈 / **02** 离散输入 / **03** 保持寄存器 / **04** 输入寄存器；写 **05** 单线圈 / **06** 单寄存器（周期到点重复写、显示回显）。
  - **半双工**：一次只在途一条请求，收到响应或超时（1s）后再发下一条；每行按各自周期独立调度。
  - 状态一目了然：**OK / 超时 / 异常（从机返回的异常码）/ 响应无效 / 发送失败**。
- **传输自适应** —— 自动按连接选择：串口 → **Modbus RTU**（含 CRC），TCP Client → **Modbus TCP**（MBAP）；也可手动强制 RTU / TCP。
- **本地回显模式** —— 串口适配器会回显发出的帧（RS-485 半双工常见）时勾选：先剥掉一份与请求相同的回显（容忍前导噪声 / 残留字节）再解析真实响应。尤其写 05/06 的回显与「写成功」同形，不开会把回显当成功、丢掉从机的异常。
- **稳健的响应核对** —— 严格校验从机ID / 返回数量 / 写回显地址值 / 异常功能码匹配；RTU 坏帧「丢 1 字节重同步」（不清整缓冲，杜绝本地回显粘包把响应一起丢掉），TCP 按 MBAP 长度自分帧。
- 新增纯逻辑 `src/modbus_master.py` + 对话框 `src/modbus_master_dialog.py` + `tests/test_modbus_master.py`（34 项）；三语 i18n；**无新依赖**；启用前对原有功能零影响；经**三轮对抗式评审**收敛。

> 提示：不要同时开启「自动应答 · Modbus 从机」与「Modbus 主机」——一个收请求、一个发请求，混用会互相干扰（详见对话框内「?」说明）。
