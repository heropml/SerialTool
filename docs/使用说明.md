## 1. 快速上手

1. 桌面双击 **CommTool** 图标启动
2. 左侧"连接设置"里选**类型**（串口或网络）→ 填串口参数 / 网络地址端口 → 点 **打开串口 / 打开 / 连接 / 开始监听**
3. 右侧"数据区"实时显示收发内容，下方"发送区"输入要发送的数据

## ★ CommTool v1.2.5 多窗口独立配置

现在可以同时开多个独立窗口，各调各的设备、互不干扰：

- **新建窗口** — 「帮助 → 新建窗口」再开一个独立窗口（也可直接再运行一次程序）；标题带 (2)/(3) 区分。
- **配置互不覆盖** — 每个窗口用各自的配置文件（主窗口 settings.ini，其余 settings-2.ini…），改设置、关窗口不再互相覆盖；靠文件锁自动分配槽位，连"双击开两个"也自动隔离。
- **各自独立** — 每个窗口有独立的连接、收发、自动应答、Modbus、终端等状态，互不干扰（同一个串口仍只能被一个窗口打开）。
- **多显示器兜底** — 窗口若落到已断开的屏幕外，自动搬回主屏，避免"进程在、窗口看不见"。

## ★ CommTool v1.2.4 终端模式

「发送区」设置里新增「终端模式」开关——把发送框变成一个轻量串口终端，适合登录 Linux 串口控制台敲命令调试：

- **逐字符即时发送** — 在发送框直接键入，每个按键立即发给设备：回车 / Backspace / Tab / Ctrl+C / 方向键（↑↓ shell 历史、←→ 移动）/ ESC / Home / End / Delete 等都透传。
- **终端语义渲染** — 数据区按终端方式显示设备回显：正确处理退格擦除、`\r` 回车、光标移动，以及行编辑常用的 ANSI 序列（`ESC[J` / `ESC[K` 擦除、`ESC[C` / `ESC[D` 光标）；颜色码 `ESC[..m` 自动忽略（带颜色的 `ls` 显示干净、不再有 `^[[31m` 乱码）。
- **本地回显 / 回车** — 「本地回显」开关（设备不回显时用）；「回车」可选 CR / LF / CRLF（默认 CR，适配多数 Linux 控制台）。
- 开终端模式时，用不上的格式设置（HEX 显示 / 时间戳 / 分包 / HEX 发送 / 校验等）自动淡化禁用、并停掉定时发送；关闭后恢复。
- 局限：不解析全屏 TUI（`vi` / `top` 等满屏定位），那些仍用专门终端。

## ★ CommTool v1.2.3 串口稳健性与列宽记忆

- **串口选择防乱跳** — 选中的串口被后台扫描偶发瞬时掉枚举时不再跳到别的口；口长期不在则自动移除「未检测到」占位、回落第一个真实口（不再常驻残留项）。
- **串口被拔出自动断开** — 已连接的串口连续几次检测不到（拔出 / 掉驱动）→ 自动断开并提示「串口 X 已移除」，不再卡在"已连接"假象。
- **串口掉线不自动重连** — 串口断开多为拔出 / 断电的物理事件，重连无意义且可能连错口，故串口掉线后干净断开、不重连（网络 TCP / UDP 仍自动重连）。
- **对话框列宽记忆** — 「多条发送」「Modbus 主机轮询」「自动应答」拖动调整的列宽现在持久化，重开 / 重启保持，并随会话配置档导入导出。

## ★ CommTool v1.2.1 Modbus 主机增强

在 v1.2.0「Modbus 主机轮询」基础上补齐写多功能码、加固误操作安全、表格可拖列——经六轮对抗式评审收敛：

- **写多功能码** — 主机写补上 **0F 写多线圈 / 10 写多寄存器**：在「数量/写值」格填多个值（逗号或空格分隔，如 `100,200,300`；0F 线圈填 0/1），数量由值个数决定。
- **误操作安全**：非法/越界的 **从机ID(0..247)、起始地址(0..65535)、写值** 一律**拒发**而非静默钳位（杜绝非法 ID 变广播地址 0 误写所有从机）；写多任一项非法或超规范上限 → **整组拒发**（不静默截断、不让后续值前移到错地址）；请求超时后短暂**静默期**，避免迟到响应被误配对到下一行；主机轮询激活时跳过自动应答，防与「自动应答·Modbus 从机」互相把响应当请求回发。
- **草稿编辑** — 启用轮询时编辑规则只进草稿、不再自动提交发送，须点**「应用」**才生效（防把读改成写时误发）；未应用草稿关窗重开保留。
- **可拖拽列** — 名称~值各列之间可**左右拖动调宽**，所有行与表头像素同步；功能码列默认更窄。
- 健壮性：单条规则损坏只跳过不清空整表；字符串 `"false"` 正确识别为停用；大量非法规则异步排程不再递归崩溃。关闭 Modbus 主机时行为与 v1.2.0 一致。
- **互斥 + 连接自愈**：自动应答与 Modbus 主机**一键互斥**（开一个自动关另一个、两个对话框开关同步）；TCP/串口半帧写入即时断流重连、不污染后续帧；Modbus 地址/数值输入兼容前导零（如 `08`/`010`）。

## ★ CommTool v1.2.0 Modbus 主机轮询

继自动应答的 Modbus RTU **从机**之后，新增 **主机（master / 轮询）**：把本工具当 Modbus 主机，按每行周期轮询从机寄存器 / 线圈并实时显示结果——主机联调、查表、巡检不用再手敲请求帧：

- **Modbus 主机** — 数据区标题栏点「Modbus 主机」，每行配 `名称 / 从机ID / 功能码 / 起始地址 / 数量(读)或写值(写) / 周期ms` → 实时 **值（十进制 + 十六进制）** + **状态**。读 `01` 线圈 / `02` 离散输入 / `03` 保持 / `04` 输入寄存器；写 `05` 单线圈 / `06` 单寄存器（周期到点重复写、显示回显）。
- **半双工** — 一次只在途一条请求，收响应或超时（1s）后再发下一条；每行按各自周期独立调度。状态：**OK / 超时 / 异常（从机异常码）/ 响应无效 / 发送失败**。
- **传输自适应** — 串口 → Modbus RTU（含 CRC），TCP Client → Modbus TCP（MBAP）；也可手动强制。
- **本地回显** — 串口适配器会回显发出帧（RS-485 半双工常见）时勾选：先剥掉一份与请求相同的回显（容忍前导噪声 / 残留）再解析真实响应。尤其写 `05/06` 的回显与「写成功」同形，不开会把回显当成功、丢掉从机异常。
- **稳健核对** — 校验从机ID / 返回数量 / 写回显 / 异常功能码匹配；RTU 坏帧丢 1 字节重同步（不清整缓冲），TCP 按 MBAP 自分帧。新增 `src/modbus_master.py` + 对话框 + `tests/test_modbus_master.py`（34 项）；三语 i18n；无新依赖；启用前不影响原有行为；经三轮对抗式评审收敛。
- 提示：不要和「自动应答 · Modbus 从机」同时开（一收一发互扰，详见弹窗内「?」）。

## ★ CommTool v1.1.9 自动应答脚本应答

自动应答新增**脚本应答**：每条规则写一段 Python，命中时动态生成应答，适配静态模板与位掩码覆盖不了的复杂逻辑（按收到内容算回值、查表、状态相关响应、自定义 CRC 等）：

- **脚本应答** — 规则行点「脚本」，定义 `reply(frame, ctx)`：返回 `bytes`=一帧 / `list[bytes]`=多帧 / 文本 / `None`=不回；脚本拥有整帧、自己用 ctx 算校验。
- **ctx** — `.state` 当前状态 · `.seq` 自增序号 · `.hits` 命中数；**可定制 CRC** `ctx.crc(data, width, poly, init, refin, refout, xorout, byteorder)` + 便捷 `crc16`(Modbus) / `crc8` / `sum8` / `xor8` / `hexbytes` / `tohex`。
- 编辑器内置「测试」跑一帧看输出；「启用脚本」开关保留代码而临停；启用时该行静态「回复 / 校验」置灰让位。
- **安全隔离** — 脚本独立子进程执行；超时（默认 1s）整组强制终止（含脚本起的子进程）、不冻结 GUI；预览 / 测试与实发各用独立进程互不污染；导入含脚本的配置先确认信任（拒绝则清空脚本）。
- 与 v1.1.8 位掩码叠加；故障注入 + 延时 + 状态机 goto 对脚本同样生效；三语 i18n；新增 `tests/test_script.py`；无新依赖；脚本默认空 = 不启用，关时与 v1.1.8 一致。

## ★ CommTool v1.1.8 自动应答位掩码与字段匹配

自动应答「匹配」做到**子字节粒度**：除整字节 `??` 通配，现在还能按半字节 / 位 / 字段匹配：

- **位掩码 / 字段级** — 匹配从「逐字节精确 / 整字节通配」升级为逐字节 `(值, 掩码)` 比对，命中需 `(收到字节 & 掩码) == (值 & 掩码)`：
  - 整字节：精确 `AB`；通配 `??` / `XX`（不变）
  - 半字节通配：`A?`（高 4 位 = A）、`?5`（低 4 位 = 5），`X` 同 `?`
  - 位级掩码：`b:` + 8 位 `0/1/x`，`x`=该位不关心，如 `b:1xxxxxx1` 只看最高与最低位
  - 混写：`AA b:1001xxxx ?5` = 字节0 须 `AA`、字节1 高 4 位 `1001` 低 4 位任意、字节2 低 4 位 `5`
- **字段级** — 「某偏移处子字段 == X」用通配填充到偏移 + 「前缀」模式（如第 3 字节 bit0 须为 1：匹配 `?? ?? ?? b:xxxxxxx1`、模式「前缀」）。
- **向后兼容** — 空格仅作分隔、纯 HEX 跨空格拼接（`A B`=0xAB）；整字节精确 / 通配解析与改前逐字节等价，旧规则零行为变化。三种模式（包含 / 相等 / 前缀）对掩码同样生效；仅 HEX 模式有意义。- 实时匹配与离线规则测试器共用解析、自动获得；三语 i18n；新增 `tests/test_match_mask.py`（17 项）；无新依赖。

## ★ CommTool v1.1.7 自动应答 Modbus RTU 从机

自动应答新增 **Modbus RTU 从机**模式：开启后整条引擎作为从机，不接真实设备也能把上位机 / 主站调通：

- **Modbus 从机** — 自动应答弹窗顶栏新增「Modbus 从机」按钮：启用 + 从机地址 + 寄存器表（按「空间 · 起始地址 · 值」配初值，地址支持 `0x` / 十进制、线圈 0/1）。开启后自动应答主机读写：
  - 读：`01` 线圈 / `02` 离散输入 / `03` 保持寄存器 / `04` 输入寄存器；
  - 写：`05` 单线圈 / `06` 单寄存器 / `0F` 多线圈 / `10` 多寄存器（写改运行态寄存器、可读回）；
  - 非法功能码 / 越界地址 / 非法数据 自动回**异常响应**（`0x80|功能码` + 异常码）。
- **稳健组帧** — RTU 无帧头，按「功能码长度 + CRC 自洽」双重判定切帧 + 跨包缓冲；CRC 不符逐字节重同步，杜绝串口噪声 / 错位后永久失步。
- **精确回请求方** — 作 TCP 服务器带多客户端时，响应精确回发给请求的那个客户端（每客户端独立半包缓冲，断开自动清）。
- **故障注入仍生效** — 全局丢包 / 错 CRC / 错长度照样作用于 Modbus 响应，压测主站重传与容错。
- 开启 Modbus 时规则 / 状态机 / 帧头组帧让位，弹窗自动收起这些区块、只留故障注入 + 醒目提示条，消除「看着启用却不响应」。
- 另修暗色主题下复选框看不出选中 / 未选中。新增 `tests/test_modbus_slave.py`（仓库首个自动化测试）；Modbus 默认关闭，关时与 v1.1.6 行为一致。三语同步。

## ★ CommTool v1.1.6 自动应答多步状态机

自动应答新增**多步状态机**，把多条规则串成按帧序列推进的握手 / 会话：

- **状态机框** — 弹窗新增「状态机」框：启用 / 初始状态 / 当前状态实时显示 / 重置。每条规则可选两个字段：
  - **仅状态（when）** — 当前状态等于它（可逗号分隔多个、状态名可含空格、留空 = 任意状态）时才命中；
  - **跳转（goto）** — 该规则应答发出后把当前状态切到它。

  实现「先收 A 才应答 B」「不同握手阶段回不同帧」等按序流程。when/goto 两列仅状态机启用时显示。
- **测试器同步** — 离线测试器显示当前状态、命中后跳转到的状态，及「内容命中但状态不符」提示（不发送、不推进）。
- **稳健推进** — goto 仅在应答**真正发出**后推进（延时未到 / 断连 / 无客户端 / 发送失败 / 丢包都不推进），主机重试仍命中当前状态。
- **并发序列化** — 带 goto 的应答整条（多段 `|` + 段间延时）发完前，后续帧进有界 FIFO 按收帧顺序处理，杜绝随机延时下状态乱序。
- 断连 / 重连 / 切总开关 / 导入配置 / 手动重置回到初始状态；编辑规则不打断进行中的握手。状态机默认关闭，关时与 v1.1.5 行为一致。三语同步。

## ★ CommTool v1.1.5 自动应答五项增强

自动应答从"能用"推到"好用 + 可压测"：

- **校验段（内层 / 额外校验）** — 每条应答可加多段：对帧 `[起始..结束]` 算校验、覆盖到指定偏移或追加，在行尾「校验」之前按序计算。配合 `{rN}` 占位符回显，内层 CRC 随回显字段**自动重算**（解决「外层 Sum + 内层 CRC」硬编码）。每行「校验段」按钮进入。
- **规则测试器（离线）** — 顶栏「测试」：输入一帧 HEX → 看命中哪条规则 + 应答预览（占位符替换 + 校验都算好），不发送、不计数。
- **命中统计** — 每条规则行实时「命中 N」+ 顶栏「重置统计」。
- **故障注入（全局，压测主机）** — 弹窗「故障注入」框：按概率 丢包（测重传）/ 错CRC（测校验）/ 错长度（测分帧）；被注入帧在数据区有 `⚠` 标记。
- **范围延时** — 应答「延时」字段支持 `100`（固定）或 `100-300`（每次发随机抖动，模拟设备 turnaround）。
- 「帧头+长度组帧」「故障注入」两框右侧加一句话说明 + 「?」例子帮助窗。

## 5. 主要功能

### 5.1 连接设置

左上角 **「连接设置」** 卡片，第一行是 **「类型」** 下拉，串口 + 网络共 5 个选项（新装默认 **Serial 串口**）：

| 类型 | 说明 |
|------|------|
| **Serial** | 串口（COM 口）收发 |
| **UDP** | 普通 UDP 收发 |
| **UDP Multicast** | UDP 组播 |
| **TCP Server** | 作为 TCP 服务端监听 |
| **TCP Client** | 作为 TCP 客户端连接 |

卡片里的字段会**随类型动态显隐**：

- **Serial（串口）**
  - **端口**（下拉本机串口，旁边 **⟳** 手动刷新；后台也会定时扫描热插拔）
  - **波特率**（可编辑下拉，常用 1200~2000000）
  - **数据位**（5 / 6 / 7 / 8，默认 8）、**校验位**（None / Even / Odd / Mark / Space）、**停止位**（1 / 1.5 / 2）
  - 点 **「打开串口」** 连接
- **UDP**
  - **本地IP**（下拉，列出本机网卡 IP，`0.0.0.0` = 所有网卡）
  - **本地端口**
  - **指定远程** 开关 — **关** = 回复最近发来数据的对端；**开** = 启用下面的 **远程IP / 远程端口**，固定发往该地址
    - **关**模式下每收到一包数据，会把灰显的「远程IP / 远程端口」自动刷成**最近发来方的地址**（让你看到当前在跟谁通信、发送会回复给谁；对端变化才刷）。收到某方的包后想固定回它，直接打开「指定远程」即预填好该地址
- **UDP Multicast（组播）**
  - **本地IP**（出 / 入网卡，`0.0.0.0` = 默认）
  - **组播地址**（如 `239.0.0.1`，须在 `224.0.0.0 ~ 239.255.255.255` 范围内）
  - **本地端口**
- **TCP Server**
  - **本地IP**、**本地端口** → 点 **「开始监听」**
  - 有客户端连入后出现 **「目标」** 下拉，可选某个客户端或 **「全部」**（广播）来发送
- **TCP Client**
  - **远程IP**（必填）、**远程端口**（必填）→ 点 **「连接」**

**动作按钮**文案随协议 + 状态变化：

| 类型 | 按钮 |
|------|------|
| Serial | 打开串口 / 关闭串口 |
| UDP / UDP Multicast | 打开 / 关闭 |
| TCP Server | 开始监听 / 停止监听 |
| TCP Client | 连接 / 断开 |

连接建立后整张卡片**锁定变灰**；被禁用的输入框也会变灰。

### 5.2 数据接收

收发同框显示，箭头区分方向：

| 标记 | 含义 |
|------|------|
| `← 收` | 接收到的数据 |
| `→ 发` | 发送出去的数据 |

**滚动锁定 / 自动跟随**

- 往上翻历史时视图**定住不动**，新数据照常在后台接收、不丢
- 右下角浮出 **「↓ 最新」** 按钮，点一下回到底部并恢复自动跟随

**单击行高亮**

- 单击数据区任意一行**高亮该行**，再点一次取消，方便盯住关心的那一帧

**显示选项**（数据区卡片）

- **HEX 显示** — 切换 16 进制 / 文本（UTF-8 优先，中文乱码自动回退 GBK）；收 / 发显示格式都只由此开关决定，与「HEX 发送」无关
- **自动换行** — 行超长自动折行，关闭后右侧滚动条
- **字符编码** — 选 自动 / UTF-8 / GBK / GB2312 / GB18030 / Big5 / ASCII / Latin-1 共 8 项；影响 RX 解码 / TX 文本编码 / 文件加载
- **显示时间戳** — 每个新块前加 `[2026/06/03 09:48:54 023]`，同时显示 ← / → 方向箭头
- **时间分包** + **超时 ms** — 两次接收间隔超过设定毫秒就开新行（默认 20ms，适合 Modbus）
- **换行分包** + **换行符模式** — 按 `\r\n` / `\r` / `\n` 切行
  - **自动**：三种风格全识别（Windows + Linux + Mac 兼容）
  - **CRLF / LF / CR**：只认指定换行符，严格模式
- **实时记录** — Xshell 风格连续日志写到 `.log` 文件，**显示什么写什么**（带时间戳、箭头、HEX/ASCII）
  - **按大小分包（v1.0.2）** — 开关旁的下拉可选 不分包 / 1M / 2M / 5M / 10M / 20M / 50M / 100M，也可手填自定义（如 `3M`、`500K`）；写满设定大小自动切到新文件（原名加 `_001` / `_002`…）
  - 当前日志文件路径会显示在**底部状态栏右侧**（详见 [7 状态栏](#7-状态栏)）
- **最大行数** — 显示行上限，超出自动丢弃最旧的（防止长跑卡顿）；**不影响日志文件**
- **只显高亮行** — 打开后只保留命中关键字的行，其余隐藏（配合「关键字高亮」用，详见 [5.4](#54-关键字高亮)）

**字号** — 右上角 `A−` `A+` 微调（7~28pt），适配大屏 / 小屏

**保存 / 清空** — 卡片底部按钮，保存到 `save_log_YYYYMMDD_HHMMSS.log`

### 5.3 数据发送

- **HEX 发送** — 输入的 16 进制字符串按字节发送，下面写法都识别：

| 写法 | 解析 |
|------|------|
| `AA BB CC DD` | 4 字节 |
| `AABBCCDD` | 4 字节 |
| `AA-BB-CC-DD` | 4 字节 |
| `AA:BB:CC:DD` | 4 字节 |
| `0xAA 0xBB 0xCC` | 3 字节 |
| `// 注释\nAA BB` | 2 字节 |
| `/* 段注释 */ AA BB` | 2 字节 |

  非法字符（如 `AA ZZ BB`）会拒绝发送并提示格式错误。

- **追加换行** + **换行符模式（CRLF / LF / CR）** — 每次发送自动追加换行符（AT 命令场景常用）
- **定时发送** + **周期 ms** — 周期性自动发送当前内容，最小 10ms；发送失败时（未连接 / 数据格式错误 / 无目标等）自动停止
- **校验**（10 种算法）— 发送前在末尾追加校验字节

| 算法 | 字节 | 用途 |
|------|------|------|
| 无 | 0 | 不追加 |
| 和校验 / ADD8 | 1 | 简单累加和 |
| 累加和取反 / ~ADD8 | 1 | 累加和按位取反 |
| 异或 / XOR8 | 1 | 全字节异或 |
| CRC8 | 1 | 多项式 0x07（标准 CCITT）|
| **MOBUS** | 1 | **CRC8 多项式 0x31**，国内嵌入式常用 |
| ModbusCRC16 | 2 | Modbus RTU 标准，小端 |
| CCITT-CRC16 | 2 | CRC-16/CCITT-FALSE，大端 |
| CRC32 | 4 | Ethernet / ZIP 标准 |
| ADD16 | 2 | 16 位累加和 |

**读取文件** — 直接把文件内容塞进发送框（HEX 模式自动转 hex 字符串）

**多条发送** — 发送区「多条发送」按钮，多行数据批量管理，详见 [5.5](#55-多条发送)

**快捷栏（v1.0.2）** — 发送区上方一行直接平铺当前分组里的每条命令为按钮，点一下即发，无需打开弹窗，详见 [5.5](#55-多条发送)

### 5.4 关键字高亮

数据区标题栏 **「关键字高亮」** 按钮打开弹窗，给数据区里命中的关键字上色，一眼定位关心的字段：

- **多条关键字**，每条**独立颜色**
- 着色方式可选 **背景** 或 **文字**
- 可限定范围：**收 / 发 / 收发**
- **区分大小写**子串匹配；自动跳过时间戳，不会误高亮
- 规则**持久化**，下次启动保留

**分组 / 规则集（v1.0.2）**

- 弹窗左侧是**分组列表**，可 **新建** / **双击重命名** / **删除** 分组，把不同场景的关键字归到各自分组
- 数据区标题栏新增**分组下拉**，选哪个分组生效；含 **「（关闭）」** 项可一键停用全部高亮
- 正在弹窗里**编辑的分组**与当前**生效的分组**相互独立，改这个不影响那个

配合数据区标题栏的 **「只显高亮行」** 可切换按钮（开启时高亮），可只保留命中关键字的行、隐藏其余。

### 5.5 多条发送

发送区 **「多条发送」** 按钮打开弹窗，一次管理多条待发数据：

- **每行一条数据 + 复选框**
- 每行可独立设 **HEX / 换行 / 校验**，并可填 **名称** 和 **延时(ms)**（v1.0.2 新增列）
- 勾选多条后点 **「循环发送」**，按每行各自的「延时」依次轮发
- 条目**持久化**，下次启动保留

**分组（v1.0.2）**

- 弹窗左侧是**分组列表**，可 **新建** / **删除** / **双击改名**，把常用指令按场景归类

**主界面快捷栏（v1.0.2）**

发送区上方一行就能直接用，不必每次开弹窗：

- 一行 = **[多条发送(编辑)]** + **[▶循环]** + **[分组下拉]**，后面平铺**当前分组里每条命令的按钮**
- 点某条命令按钮**直接发**这条
- 点 **[▶循环]** 按每行各自的「延时」依次轮发该分组的命令

## 6. 实用技巧

- **Modbus 调试**：开 HEX 显示 + HEX 发送 + 时间分包（20ms）+ ModbusCRC16
- **AT 命令调试**：关 HEX 显示 + 追加 CRLF + 换行分包（自动）
- **长跑监控**：开实时记录 → 日志文件可后续 Notepad++ / VS Code 打开分析
- **HEX 写注释**：HEX 发送框可写 `// 行注释`、`/* 段注释 */`、`# 行注释`，提交时自动去除
- **拖动布局**：左右 / 上下分割条都可以拖，调成顺手的比例后下次启动保留
- **盯关键帧**：给关键字配高亮 + 开「只显高亮行」，刷屏里也能只盯命中的行；翻历史时滚动锁定不怕被新数据冲走
- **批量指令**：常用的几条指令丢进「多条发送」勾上循环，按每行延时轮发，省得一条条手敲
- **快捷直发**：把常用命令归到一个分组，发送区上方快捷栏一键平铺成按钮，点哪条发哪条，不用开弹窗
- **分场景高亮**：不同协议 / 模块各建一个高亮分组，用数据区的分组下拉随时切换，调试别的设备时选「（关闭）」即可
- **长跑切包**：实时记录选个分包大小（如 10M），日志写满自动切新文件，状态栏随时能看到当前写到哪个文件

## 8. 常见问题

**Q：TCP Server 开始监听失败 / 端口被占用？**
A：该端口被其他程序占用，或需要权限；换个端口或关掉占用程序再试。

**Q：TCP Client 连不上？**
A：检查远程 IP / 端口是否正确、对端是否在监听、防火墙是否放行。

**Q：UDP 点发送提示"无可发送目标"？**
A：没开"指定远程"且还没收到过任何对端数据。开启"指定远程"填好远程地址，或等对端先发一包。

**Q：UDP 组播收不到数据？**
A：确认防火墙放行、收发在同一组播地址和端口、在同一网段 / 网卡。

**Q：定时发送不工作？**
A：周期最小 10ms，且必须保持连接；断开时定时会自动暂停。

**Q：中文显示乱码？**
A：自动模式默认 UTF-8 优先，乱码会自动回退 GBK。如果还是不对，设备可能用其他编码（如 Big5），在"字符编码"下拉里明确指定，或切到 HEX 显示看原始字节。

**Q：HEX 发送报"长度必须为偶数"？**
A：HEX 是按字节解析的，`AA B` 是 3 个 hex 字符无法配对，补成 `AA 0B` 或 `AAB0` 即可。

**Q：日志文件大了会卡吗？**
A：写入是追加模式，几十 MB / 几百 MB 都不会卡。"最大行数"限制只影响**界面显示**的行数，不影响日志写入。

**Q：往上翻历史时新数据会丢吗？**
A：不会。滚动锁定只是把视图定住，数据照常在后台接收。点右下角「↓ 最新」回到底部即可继续自动跟随。

**Q：开了「只显高亮行」却什么都没显示？**
A：说明当前没有命中的行。先在「关键字高亮」里配好关键字，且范围（收 / 发 / 收发）要匹配，命中的行才会留下。
