## 1. 快速上手

1. 桌面雙擊 **CommTool** 圖示啟動
2. 左側「連線設定」裡選**類型**（串口或網路）→ 填串口參數 / 網路位址埠，點 **開啟串口 / 開啟 / 連線 / 開始監聽**
3. 右側「資料區」即時顯示收發內容，下方「傳送區」輸入要傳送的資料

## ★ CommTool v1.1.0 收發統計

底部狀態列的 RX/TX 從「純位元組數」升級為**位元組 · 封包數 · 即時速率**：

- **封包計數** — RX 每到達一塊資料計 1 包，TX 每成功傳送一次計 1 包。
- **即時速率** — 1Hz 取樣的目前收/發速率（B/s）；常駐統計，斷線後自然歸零。
- **懸停看明細** — 滑鼠停在 RX/TX 標籤上彈出 tooltip：總量 / 封包數 / 目前速率 / **峰值速率** / 錯誤數（錯誤數 >0 時標籤上追加 ⚠ 標記）。
- **重置統計** — 在狀態列上**按右鍵 → 「重置統計」**清零所有計數器（不動資料區，跟隨語言/主題）。
- **macOS 修復** — 修正了 macOS 上 tooltip 背景透明、文字看不清的問題（改用自繪不透明提示框）。

## 2. 主要功能

### 2.1 連線設定

左上角「連線設定」卡片，第一行 **「類型」** 下拉提供串口 + 網路共 5 個選項（新裝預設 **Serial 串口**）：

| 選項 | 說明 |
|------|------|
| **Serial** | 串口（COM 埠）收發 |
| **UDP** | 無連線收發 |
| **UDP Multicast** | UDP 群播 |
| **TCP Server** | 作為伺服器監聽，接受用戶端連入 |
| **TCP Client** | 作為用戶端連向遠端伺服器 |

下方欄位**隨類型動態顯隱**：

- **Serial（串口）**
  - **串口**（下拉本機串口，旁邊 **⟳** 手動刷新；後台也會定時掃描熱插拔）
  - **鮑率**（可編輯下拉，常用 1200~2000000）
  - **資料位元**（5 / 6 / 7 / 8，預設 8）、**校驗位元**（None / Even / Odd / Mark / Space）、**停止位元**（1 / 1.5 / 2）
  - 點 **「開啟串口」** 連線
- **UDP**
  - **本地IP**（下拉，列出本機網卡 IP，`0.0.0.0` = 所有網卡）
  - **本地埠**
  - **指定遠端**開關 — 關＝回覆最近發來資料的對端；開＝啟用下面的 **遠端IP / 遠端埠**，固定發往該位址
    - **關**模式下每收到一包資料，會把灰顯的「遠端IP / 遠端埠」自動刷成**最近發來方的位址**（讓你看到目前在跟誰通訊、傳送會回覆給誰；對端變化才刷）。收到某方的封包後想固定回它，直接開啟「指定遠端」即預填好該位址
- **UDP Multicast（群播）**
  - **本地IP**（出 / 入網卡，`0.0.0.0` = 預設）
  - **群播位址**（如 `239.0.0.1`，須落在 `224.0.0.0` ~ `239.255.255.255`）
  - **本地埠**
- **TCP Server**
  - **本地IP**、**本地埠** →「開始監聽」
  - 有用戶端連入後出現 **「目標」** 下拉，可選某個用戶端或 **「全部」**（廣播）來傳送
- **TCP Client**
  - **遠端IP**（必填）、**遠端埠**（必填）→「連線」

**動作按鈕**文案隨類型 + 狀態變化：

- Serial：開啟串口 / 關閉串口
- UDP / UDP Multicast：開啟 / 關閉
- TCP Server：開始監聽 / 停止監聽
- TCP Client：連線 / 中斷

連線建立後整張卡片**鎖定變灰**；隨協議停用的輸入框也會變灰。

### 2.2 資料接收

收發同框顯示，箭頭區分方向：

| 標記 | 含義 |
|------|------|
| `← 收` | 接收到的資料 |
| `→ 發` | 傳送出去的資料 |

**顯示選項**（資料區卡片）

- **HEX 顯示** — 切換 16 進位 / 文字（UTF-8 優先，中文亂碼自動回退 GBK）；收發顯示格式**只由本開關決定**，收 / 發一致，與「HEX 傳送」開關無關
- **自動換行** — 行超長自動換行，關閉後右側捲動條
- **字元編碼** — 選 自動 / UTF-8 / GBK / GB2312 / GB18030 / Big5 / ASCII / Latin-1 共 8 項；影響 RX 解碼 / TX 文字編碼 / 檔案載入
- **顯示時間戳** — 每個新區塊前加 `[2026/06/03 09:48:54 023]`，同時顯示 ← / → 方向箭頭
- **時間分包** + **逾時 ms** — 兩次接收間隔超過設定毫秒就開新行（預設 20ms，適合 Modbus）
- **換行分包** + **換行符模式** — 按 `\r\n` / `\r` / `\n` 切行
  - **自動**：三種風格全識別（Windows + Linux + Mac 相容）
  - **CRLF / LF / CR**：只認指定換行符，嚴格模式
- **即時記錄** — Xshell 風格連續日誌寫到 `.log` 檔案，**顯示什麼寫什麼**（帶時間戳、箭頭、HEX/ASCII）
  - **按檔案大小分包** — 開關旁下拉可選 不分包 / 1M / 2M / 5M / 10M / 20M / 50M / 100M，也可手填自訂（如 `3M`、`500K`）；寫到設定大小自動切到新檔（原名加 `_001`/`_002`…）
- **最大行數** — 顯示行上限，超出自動丟棄最舊的（防止長跑卡頓）；**不影響日誌檔案**
- **只顯高亮行** — 開啟後只保留命中關鍵字高亮規則的行，其餘隱藏；方便在大量資料裡只盯關鍵內容

**捲動鎖定 / 自動跟隨**

- 資料停在底部時自動跟隨最新資料；一旦**往上捲**，畫面就定住不再被新資料頂走，但新資料**照常接收**、不會漏
- 右下角浮動 **「↓ 最新」** 按鈕：點一下回到底部並恢復自動跟隨

**單擊高亮**

- 單擊資料區任一行即高亮整行，方便鎖定一筆關鍵收 / 發；再點一下取消

**字號** — 右上角 `A−` `A+` 微調（7~28pt），適配大螢幕 / 小螢幕

**儲存 / 清空** — 卡片底部按鈕，儲存為 `save_log_YYYYMMDD_HHMMSS.log`

**右鍵選單** — 在資料區按右鍵彈出 複製 / 全選 / 清空 / 儲存，選單文字跟隨程式語言（而非系統語言的 Qt 預設選單）

### 2.3 資料傳送

- **HEX 傳送** — 輸入的 16 進位字串按位元組傳送，下面寫法都識別：

| 寫法 | 解析 |
|------|------|
| `AA BB CC DD` | 4 位元組 |
| `AABBCCDD` | 4 位元組 |
| `AA-BB-CC-DD` | 4 位元組 |
| `AA:BB:CC:DD` | 4 位元組 |
| `0xAA 0xBB 0xCC` | 3 位元組 |
| `// 註解\nAA BB` | 2 位元組 |
| `/* 區塊註解 */ AA BB` | 2 位元組 |

  非法字元（如 `AA ZZ BB`）會拒絕傳送並提示格式錯誤。

- **追加換行** + **換行符模式（CRLF / LF / CR）** — 每次傳送自動追加換行符（AT 命令場景常用）
- **定時傳送** + **週期 ms** — 週期性自動傳送目前內容，最小 10ms；傳送失敗時（未連線 / 資料格式錯誤 / 無目標等）自動停止
- **校驗**（10 種演算法）— 傳送前在末尾追加校驗位元組

| 演算法 | 位元組 | 用途 |
|------|------|------|
| 無 | 0 | 不追加 |
| 和校驗 / ADD8 | 1 | 簡單累加和 |
| 累加和取反 / ~ADD8 | 1 | 累加和按位取反 |
| 異或 / XOR8 | 1 | 全位元組異或 |
| CRC8 | 1 | 多項式 0x07（標準 CCITT）|
| **MOBUS** | 1 | **CRC8 多項式 0x31**，國內嵌入式常用 |
| ModbusCRC16 | 2 | Modbus RTU 標準，小端 |
| CCITT-CRC16 | 2 | CRC-16/CCITT-FALSE，大端 |
| CRC32 | 4 | Ethernet / ZIP 標準 |
| ADD16 | 2 | 16 位元累加和 |

**讀取檔案** — 直接把檔案內容塞進傳送框（HEX 模式自動轉 hex 字串）

**多條發送** — 傳送區「多條發送」按鈕開彈窗，一次管理多條常用資料、可勾選後迴圈輪發，詳見 [2.5 多條發送](#25-多條發送)

### 2.4 關鍵字高亮

資料區標題列 **「關鍵字高亮」** 按鈕開彈窗，按關鍵字替指定的收 / 發內容自動上色：

- **多條關鍵字**，每條可設**獨立顏色**
- 每條可選 **「背景」** 或 **「文字」** 上色方式
- 每條可限定生效**範圍**：收 / 發 / 收發
- 子字串比對，**區分大小寫**；自動跳過時間戳，不會誤把時間戳染色
- 規則**持久化**，下次啟動沿用

**分組／規則集** — 同一批關鍵字可整理成多個分組，按場景隨時切換：

- 彈窗左側為**分組列表**，可**新增** / **重新命名（雙擊）** / **刪除**分組
- 資料區標題列的**分組下拉**選擇哪個分組生效，含 **「（關閉）」** 項一鍵停用全部高亮
- 在彈窗裡**編輯的分組**與目前**生效的分組**相互獨立，編輯時不影響目前顯示

搭配資料區標題列的 **「只顯高亮行」** 可切換按鈕（開啟時高亮），可只保留命中關鍵字的行、隱藏其餘，從大量資料裡快速聚焦關鍵內容。

### 2.5 多條發送

傳送區 **「多條發送」** 按鈕開彈窗，集中管理多條常用資料：

- 每行一條資料，前面帶**核取方塊**
- 每行可**獨立設定** HEX / 換行 / 校驗，並可填 **「名稱」** 與 **「延時（ms）」**
- 勾選多條後 **「迴圈發送」** 按設定間隔依次輪發
- 條目**持久化**，下次啟動沿用

**分組** — 彈窗左側為**分組列表**，可**新增** / **刪除** / **雙擊改名**，把不同場景的常用命令分開管理。

**主介面快捷列** — 不必每次開彈窗，發送區上方一行直接操作：

- 一行 = `[多條發送（編輯）]` `[▶迴圈]` `[分組下拉]` ＋ 該分組每條命令平鋪成按鈕
- 點任一命令按鈕**直接發送**那條
- 按 **`▶迴圈`** 依目前分組各行的 **「延時」** 依次輪發

系統匣圖示**右鍵選單**新增 **「關於」** 入口，開啟 **「關於」對話方塊**：裡面顯示應用圖示、名稱、目前版本號、簡介，以及一個 **「檢查更新」** 按鈕。

- **檢查更新** — 點擊後自動從更新來源讀取最新版本，並與目前版本比對：
  - **有新版本** → 對話方塊顯示新版本號、**更新說明** 與 **「下載並更新」** 按鈕。點擊開始下載（帶**進度百分比**），下載完成後彈出**正常安裝精靈**，由你手動點「下一步 / 安裝」完成升級（**不是靜默安裝**）。

- **隨時取消** — 下載過程中關閉對話方塊會**自動取消下載**。

### 2.12 設定自動儲存

退出時自動存到安裝目錄的 `settings.ini`，下次啟動**完全還原**：

- 視窗位置 + 大小（含最大化狀態）
- 分隔條位置
- 目前語言 + 主題
- 所有開關 / 輸入框 / 下拉選項

安裝目錄唯讀時（比如 Program Files 裝的非管理員執行），設定自動回退到 `%APPDATA%\CommTool\settings.ini`。

## 3. 實用技巧

- **Modbus 除錯**：開 HEX 顯示 + HEX 傳送 + 時間分包（20ms）+ ModbusCRC16
- **AT 命令除錯**：關 HEX 顯示 + 追加 CRLF + 換行分包（自動）
- **長跑監控**：開即時記錄 → 日誌檔案可後續 Notepad++ / VS Code 開啟分析
- **HEX 寫註解**：HEX 傳送框可寫 `// 行註解`、`/* 區塊註解 */`、`# 行註解`，提交時自動移除
- **拖曳版面**：左右 / 上下分隔條都可以拖，調成順手的比例後下次啟動保留

## 5. 常見問題

**Q：TCP Server 開始監聽失敗 / 埠被佔用？**
A：該埠被其他程式佔用，或需要權限；換個埠或關掉佔用程式。

**Q：TCP Client 連不上？**
A：檢查遠端 IP / 埠是否正確、對端是否在監聽、防火牆是否放行。

**Q：UDP 點傳送提示「無可發送目標」？**
A：沒開「指定遠端」且還沒收到任何對端資料；開啟「指定遠端」填好遠端位址，或等對端先發一包。

**Q：UDP 群播收不到資料？**
A：確認防火牆放行、收發在同一群播位址和埠、在同一網段 / 網卡。

**Q：定時傳送不運作？**
A：週期最小 10ms，且必須保持連線；中斷時定時會自動暫停。

**Q：中文顯示亂碼？**
A：自動模式預設 UTF-8 優先，亂碼會自動回退 GBK。如果還是不對，裝置可能用其他編碼（如 Big5），在「字元編碼」下拉裡明確指定，或切到 HEX 顯示看原始位元組。

**Q：HEX 傳送報「長度必須為偶數」？**
A：HEX 是按位元組解析的，`AA B` 是 3 個 hex 字元無法配對，補成 `AA 0B` 或 `AAB0` 即可。

**Q：日誌檔案大了會卡嗎？**
A：寫入是追加模式，幾十 MB / 幾百 MB 都不會卡。「最大行數」限制只影響**介面顯示**的行數，不影響日誌寫入。
