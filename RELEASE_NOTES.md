# SerialTool v1.0.2

本次以「分组化」「主界面快捷操作」「日志可控」为主线，并做了一轮界面紧凑化与性能优化。

## ✨ 新功能

**关键字高亮 — 分组 / 规则集**
- 弹窗左侧新增**分组列表**：新建 / 双击改名 / 删除。
- 数据区标题栏的**分组下拉**选择「哪个分组生效」，含「**（关闭）**」一键停用全部高亮。
- 编辑分组与生效分组**相互独立**——可以编辑 A 组的同时让 B 组生效。

**多条发送 — 分组 + 每行延时 + 主界面快捷栏**
- 弹窗左侧新增分组列表（新建 / 删除 / 双击改名）；每行命令新增「**名称**」和「**延时(ms)**」列。
- 发送区上方新增**快捷栏**：`[多条发送(编辑)] [▶ 循环] [分组下拉]` + 该分组每条命令**平铺成按钮，点一下直接发**，不用打开弹窗。
- 循环发送按**每行各自的延时**依次轮发。

**实时记录 — 按文件大小分包**
- 实时记录开关旁新增下拉：**不分包 / 1M / 2M / 5M / 10M / 20M / 50M / 100M**，也可**手填自定义**（如 `3M`、`500K`）。
- 写满设定大小自动切到新文件（原名加 `_001` / `_002`…）。
- **当前日志文件路径**显示在底部状态栏（📝，太长中间省略、悬停看全路径）。

**数据区中文右键菜单**
- 右键弹出 **复制 / 全选 / 清空 / 保存**，跟随程序语言（不再是系统语言的 Qt 默认菜单）。

## 🔧 改进

- **界面整体紧凑化**：字号、输入框、iOS 开关、卡片间距均收紧，更接近专业串口工具的密度。
- 「**只显高亮行**」从侧边栏移到数据区标题栏，做成可切换按钮（开启时高亮），与关键字高亮控件放一起。
- 底部状态栏各项之间加**竖线分隔**，更易区分。
- 修复发送区底部按钮与发送框重叠的布局问题。
- 修复高分屏（HiDPI 缩放）下窗口边缘拖拽缩放失效。

## ⚡ 性能与稳定性

- **切换主题不再卡死**：定时发送积累大量数据后切主题，数据区重涂经批处理优化（3000 行约 0.03s）。
- 弹窗编辑**去抖**：连续输入合并为一次落盘 + 重建，条目多也不卡。
- 一批代码审查修复：迁移结果落盘（不再每次启动重复迁移）、清理旧配置键、关窗统一停止循环定时器、切分组前刷盘待提交编辑等。
