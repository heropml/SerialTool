## 1. 快速上手

1. 桌面双击 **SerialTool** 图标启动
2. 左侧"串口设置"里选端口和波特率，点 **打开串口**
3. 右侧"数据区"实时显示收发内容，下方"发送区"输入要发送的数据

## 2. v1.0.1 新功能

- **高分屏（HiDPI）缩放适配** — 界面 / 字体在高分辨率屏幕上不再过小
- **数据区滚动锁定** — 往上翻时视图定住、新数据仍正常接收；右下角浮动「↓ 最新」按钮回到底部并恢复自动跟随
- **单击高亮行** — 单击数据区某一行高亮该行（再点取消）
- **关键字高亮** — 多条关键字、各自配色，可限定收 / 发范围；规则持久化
- **只显高亮行** — 一键过滤，只保留命中关键字的行
- **多条发送** — 多行数据弹窗，每行独立设 HEX / 换行 / 校验，可循环发送
- **主题切换重涂** — 切主题时数据区已有文字按角色重涂新主题色，浅 ↔ 深切换不再有文字看不见
- **弹窗标题栏跟随主题** — 弹窗原生标题栏跟随深 / 浅主题
- **更高波特率** — 下拉新增 256000 ~ 2000000 等高速率，仍可手填自定义值
- **英文界面标签自适应** — 英文下串口设置标签宽度自适应，不再被遮挡

## 3. 主要功能

### 3.1 串口设置

- 自动检测可用 COM 口，热插拔自动刷新（每 1.5 秒后台扫描）
- 端口下拉里显示设备描述（区分多个 USB 转串口）
- 波特率：1200–921600，也可手动填任意值
- 数据位 5/6/7/8、校验 None/Even/Odd/Mark/Space、停止位 1/1.5/2

### 3.2 数据接收

收发同框显示，箭头区分方向：

| 标记 | 含义 |
|------|------|
| `← 收` | 接收到的数据 |
| `→ 发` | 发送出去的数据 |

**滚动锁定 / 自动跟随**

- 往上翻历史时视图**定住不动**，新数据照常在后台接收、不丢
- 右下角浮出 **「↓ 最新」** 按钮，点一下回到底部并恢复自动跟随

**单击行高亮**

- 单击数据区任意一行**高亮该行**，再点一次取消，方便盯住关心的那一帧

**显示选项**（数据区卡片）

- **HEX 显示** — 切换 16 进制 / 文本（UTF-8 优先，中文乱码自动回退 GBK）；收 / 发显示格式都只由此开关决定，与「HEX 发送」无关
- **自动换行** — 行超长自动折行，关闭后右侧滚动条
- **字符编码** — 选 自动 / UTF-8 / GBK / GB2312 / GB18030 / Big5 / ASCII / Latin-1 共 8 项；影响 RX 解码 / TX 文本编码 / 文件加载
- **显示时间戳** — 每个新块前加 `[2026/06/03 09:48:54 023]`，同时显示 ← / → 方向箭头
- **时间分包** + **超时 ms** — 两次接收间隔超过设定毫秒就开新行（默认 20ms，适合 Modbus）
- **换行分包** + **换行符模式** — 按 `\r\n` / `\r` / `\n` 切行
  - **自动**：三种风格全识别（Windows + Linux + Mac 兼容）
  - **CRLF / LF / CR**：只认指定换行符，严格模式
- **实时记录** — Xshell 风格连续日志写到 `.log` 文件，**显示什么写什么**（带时间戳、箭头、HEX/ASCII）
- **最大行数** — 显示行上限，超出自动丢弃最旧的（防止长跑卡顿）；**不影响日志文件**
- **只显高亮行** — 打开后只保留命中关键字的行，其余隐藏（配合「关键字高亮」用，详见 [3.4](#34-关键字高亮)）

**字号** — 右上角 `A−` `A+` 微调（7~28pt），适配大屏 / 小屏

**保存 / 清空** — 卡片底部按钮，保存到 `save_log_YYYYMMDD_HHMMSS.log`

### 3.3 数据发送

- **HEX 发送** — 输入的 16 进制字符串按字节发送，下面写法都识别：

| 写法 | 解析 |
|------|------|
| `AA BB CC DD` | 4 字节 |
| `AABBCCDD` | 4 字节 |
| `AA-BB-CC-DD` | 4 字节 |
| `AA:BB:CC:DD` | 4 字节 |
| `0xAA 0xBB 0xCC` | 3 字节 |
| `// 注释\nAA BB` | 2 字节 |
| `/* 段注释 */ AA BB` | 2 字节 |

  非法字符（如 `AA ZZ BB`）会拒绝发送并提示格式错误。

- **追加换行** + **换行符模式（CRLF / LF / CR）** — 每次发送自动追加换行符（AT 命令场景常用）
- **定时发送** + **周期 ms** — 周期性自动发送当前内容，最小 10ms；串口关闭时自动停止
- **校验**（10 种算法）— 发送前在末尾追加校验字节

| 算法 | 字节 | 用途 |
|------|------|------|
| 无 | 0 | 不追加 |
| 和校验 / ADD8 | 1 | 简单累加和 |
| 累加和取反 / ~ADD8 | 1 | 累加和按位取反 |
| 异或 / XOR8 | 1 | 全字节异或 |
| CRC8 | 1 | 多项式 0x07（标准 CCITT）|
| **MOBUS** | 1 | **CRC8 多项式 0x31**，国内嵌入式常用 |
| ModbusCRC16 | 2 | Modbus RTU 标准，小端 |
| CCITT-CRC16 | 2 | CRC-16/CCITT-FALSE，大端 |
| CRC32 | 4 | Ethernet / ZIP 标准 |
| ADD16 | 2 | 16 位累加和 |

**读取文件** — 直接把文件内容塞进发送框（HEX 模式自动转 hex 字符串）

**多条发送** — 发送区「多条发送」按钮，多行数据批量管理，详见 [3.5](#35-多条发送)

### 3.4 关键字高亮

数据区标题栏 **「关键字高亮」** 按钮打开弹窗，给数据区里命中的关键字上色，一眼定位关心的字段：

- **多条关键字**，每条**独立颜色**
- 着色方式可选 **背景** 或 **文字**
- 可限定范围：**收 / 发 / 收发**
- **区分大小写**子串匹配；自动跳过时间戳，不会误高亮
- 规则**持久化**，下次启动保留

配合数据区选项里的 **「只显高亮行」** 开关，可只保留命中关键字的行、隐藏其余。

### 3.5 多条发送

发送区 **「多条发送」** 按钮打开弹窗，一次管理多条待发数据：

- **每行一条数据 + 复选框**
- 每行可独立设 **HEX / 换行 / 校验**
- 勾选多条后点 **「循环发送」**，按设定间隔依次轮发
- 条目**持久化**，下次启动保留

### 3.6 界面

- **无边框窗口** + **iOS 风格圆角卡片** + 柔和阴影
- 左侧 sidebar（串口/数据/发送设置）+ 右侧数据区，分割条**可拖动**
- 拖动标题栏移动窗口、双击切换最大化
- 边缘原生缩放（手感和系统窗口一致）

### 3.7 多语言

标题栏左上下拉切换：**简体中文 / English / 繁體中文**，**无需重启**，所有 UI 文字瞬间切换（菜单、按钮、提示、错误消息、文件对话框全部跟着变）。

### 3.8 主题

标题栏左上紧挨语言下拉的第二个下拉切换整体配色方案，**9 个终端风主题**，影响整个窗口（侧边栏卡片、按钮、数据区、状态栏，连关闭确认对话框都跟着切）：

| 主题 | 模式 | 风格 |
|------|------|------|
| Default | 浅色 | iOS 默认 — 白卡片 + 浅灰窗口 |
| Dark | 暗色 | VSCode 通用暗灰 |
| One Half 浅色 | 浅色 | Atom 编辑器风，干净米白 |
| One Half 深色 | 暗色 | Atom 编辑器风，蓝灰调 |
| Solarized 浅色 | 浅色 | 米黄底，护眼经典 |
| Solarized 深色 | 暗色 | 深青蓝，终端经典 |
| Tango 深色 | 暗色 | Linux 风深灰 |
| Campbell | 暗色 | 漆黑（Windows Terminal 默认）|
| Ubuntu | 暗色 | 葡萄紫 |

切主题时，数据区**已有文字会按角色（时间戳 / 收 / 发）重涂成新主题色**，浅 ↔ 深切换不再有文字看不见。弹窗的原生标题栏也跟随深 / 浅主题。所选主题重启保留。

### 3.9 鼠标悬停说明

数据区/发送区每个选项把鼠标悬停在选项名上会弹出说明 tooltip — 比如鼠标停在"时间分包"上：

```
收到数据后超过下方「超时」时间无新数据就开新行
用于把短时间到达的连续数据合并显示
```

### 3.10 系统托盘

点窗口右上 `×` 弹出三选一：

| 选项 | 行为 |
|------|------|
| 最小化到托盘（默认）| 隐藏窗口到任务栏右下角 |
| 退出程序 | 真退出，保存配置 |
| 取消 | 不关 |

托盘图标：
- 单击 / 双击 → 恢复窗口
- 右键 → "显示窗口" / "退出"

### 3.11 配置自动保存

退出时自动存到安装目录的 `settings.ini`，下次启动**完全恢复**：

- 窗口位置 + 大小（含最大化状态）
- 分割条位置
- 当前语言 + 主题
- 所有开关 / 输入框 / 下拉选项
- 发送框内容、字号、最大行数

安装目录只读时（比如 Program Files 装的非管理员运行），配置自动回退到 `%APPDATA%\SerialTool\settings.ini`。

## 4. 实用技巧

- **Modbus 调试**：开 HEX 显示 + HEX 发送 + 时间分包（20ms）+ ModbusCRC16
- **AT 命令调试**：关 HEX 显示 + 追加 CRLF + 换行分包（自动）
- **长跑监控**：开实时记录 → 日志文件可后续 Notepad++ / VS Code 打开分析
- **HEX 写注释**：HEX 发送框可写 `// 行注释`、`/* 段注释 */`、`# 行注释`，提交时自动去除
- **拖动布局**：左右 / 上下分割条都可以拖，调成顺手的比例后下次启动保留
- **盯关键帧**：给关键字配高亮 + 开「只显高亮行」，刷屏里也能只盯命中的行；翻历史时滚动锁定不怕被新数据冲走
- **批量指令**：常用的几条指令丢进「多条发送」勾上循环，按周期轮发，省得一条条手敲

## 6. 常见问题

**Q：打开串口报"拒绝访问"？**
A：串口被其他程序占用（另一个串口工具、IDE 的串口监视器等），关掉那边再试。

**Q：定时发送有时不发？**
A：周期最小 10ms，且串口必须保持打开。串口断开时定时会自动暂停。

**Q：中文显示乱码？**
A：自动模式默认 UTF-8 优先，乱码会自动回退 GBK。如果还是不对，设备可能用其他编码（如 Big5），在"字符编码"下拉里明确指定，或切到 HEX 显示看原始字节。

**Q：HEX 发送报"长度必须为偶数"？**
A：HEX 是按字节解析的，`AA B` 是 3 个 hex 字符无法配对，补成 `AA 0B` 或 `AAB0` 即可。

**Q：日志文件大了会卡吗？**
A：写入是追加模式，几十 MB / 几百 MB 都不会卡。"最大行数"限制只影响**界面显示**的行数，不影响日志写入。

**Q：USB 转串口拔了没刷新？**
A：部分驱动拔插不通知系统，点端口下拉旁的 `⟳` 按钮手动刷新即可。

**Q：往上翻历史时新数据会丢吗？**
A：不会。滚动锁定只是把视图定住，数据照常在后台接收。点右下角「↓ 最新」回到底部即可继续自动跟随。

**Q：开了「只显高亮行」却什么都没显示？**
A：说明当前没有命中的行。先在「关键字高亮」里配好关键字，且范围（收 / 发 / 收发）要匹配，命中的行才会留下。
